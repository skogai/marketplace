`feature-dev` is a structured, 7-phase feature-development workflow driven by the
`/feature-dev` command and three specialized agents: explore the codebase first,
ask clarifying questions, design architecture, then implement and review for
quality — rather than jumping straight into code.

<routes>

- @README.md - full workflow philosophy and phase breakdown
- @commands/feature-dev.md - entry point command that drives the 7-phase workflow
- @agents/code-explorer.md - codebase exploration agent
- @agents/code-architect.md - architecture design agent
- @agents/code-reviewer.md - quality review agent

</routes>
