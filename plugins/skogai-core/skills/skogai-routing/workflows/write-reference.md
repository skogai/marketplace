<workflow>

<objective>
create a reference endpoint for durable concepts, rules, terminology, and tradeoffs.
</objective>

<steps>

1. name the reference after the concept it owns.
2. define the concept before listing rules.
3. keep procedural sequences out unless they clarify the concept.
4. prefer compact tables for comparisons and taxonomies.
5. link templates or scripts only when they directly implement the concept.

</steps>

<recommended_tags>

- a root concept tag matching the topic
- `<overview>`
- focused semantic child tags as needed

</recommended_tags>

<validation>

- the reference answers "what should be known?"
- the reference can be read independently.
- it does not own task sequence that belongs in a workflow.

</validation>

</workflow>
