<workflow>

<objective>
create or rewrite a routing file such as `skill.md`, `agents.md`, `claude.md`, a simple-skill template, or a nested router.
</objective>

<steps>

1. define the routing file's scope in one sentence.
2. list the endpoint types it may route to.
3. add a compact intent-to-endpoint table.
4. state the rules that must be known before loading deeper files.
5. move detailed procedures into workflows.
6. move durable explanations into references.
7. move reusable shapes into templates.
8. move repeated checks or mechanics into scripts.
9. end with success criteria.

</steps>

<recommended_tags>

- `<objective>`
- `<mental_model>`
- `<quick_start>`
- `<routing>`
- `<rules>` or a domain-specific rules tag
- `<success_criteria>`

</recommended_tags>

<validation>

- the file can be read quickly.
- it routes by intent, not by forced menu selection.
- it does not duplicate endpoint content.
- all named endpoints exist or are intentionally planned.

</validation>

</workflow>
